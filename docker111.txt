在生产环境中我们一般不会使用默认 bridge，而是创建自定义 bridge 网络，这样容器之间可以通过容器名进行 DNS 解析和服务发现。
Docker 常见有四种网络模式：bridge、host、none 和 container。
其中 bridge 是默认模式，通过虚拟网桥实现容器之间通信，并支持端口映射；
host 模式是容器直接使用宿主机网络，性能最好但没有隔离；none 模式表示容器没有网络；
container 模式是多个容器共享同一个网络空间。

在生产环境中，我们一般使用自定义 bridge 网络来实现容器之间的服务通信。
每个容器跑在不同的物理机/宿主机上，容器之间的通信就不能靠 Docker 自定义 bridge 网络了，因为 bridge 网络 只在同一台宿主机内部有效。

Overlay 网络是 Docker 提供的一种 跨主机容器通信网络。它允许：
不同物理机上的容器像在同一个局域网里一样通信。使用 容器名作为 DNS 访问服务

Overlay 网络的核心技术：

VXLAN 封装
容器网络的数据包被封装成 UDP 数据包，通过物理网络传输到另一台宿主机

Docker 内部路由
Docker 在各宿主机上部署 Swarm Node，负责将 Overlay 网络流量路由到目标容器

内部 DNS
Overlay 网络提供内置 DNS，容器可以直接用容器名访问同一 Overlay 网络里的其他容器
